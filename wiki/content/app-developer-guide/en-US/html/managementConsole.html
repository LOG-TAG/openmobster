The current version of the platform comes with a command line management tool called the Management Console. At this point in time, it provides security and account provisioning related functions. In the future, this tool will be enhanced with other management functions like device management, remote wipe, remote lock-down, tracking, etc. A GWT-based GUI interface is also on the project roadmap.

7.2. Console Commands

7.2.1. Configure

This command initializes the console for use with the Cloud Server. It registers the 'Administrator/Root' user who will be allowed to use this tool to manage the Cloud Server.

Usage: 

				
configure -a <cloudServer Ip Address> -po 1502 -u root -p <whatever root password you can remember>

Command line options:

  • -a: IP address of the Cloud Server. (Required)
  • -po: Cloud Server Port. The port out-of-the-box is 1502. (Required)
  • -u: Username to be associated with the Administrator. (Required)
  • -p: Password for this account. (Required)

Note: This is an administration account. This should not be confused with Cloud accounts provisioned to end users who will be registering their respective devices to gain mobile access to cloud services.

7.2.2. Start Cloud Administration

This command is used by the Cloud Administrator to successfully authenticate and gain access to Cloud Management functions.

Usage: 

				
startadmin -u root -p <whatever password was setup>

Command line options:

  • -u: Username that is associated with the Administrator. (Required)
  • -p: Password. (Required)

7.2.3. Register a New User

This command is used by the Cloud Administrator to provision a new user account. The user can then provision their mobile device via the CloudManager app installed on their device, using this account.

Usage: 

				
register -u <email of the user being registered> -p <whatever password that you want to associate with this account>

Command line options:

  • -u: Unique 'username' for a new account. Typically this uniqueness can be managed better by using their email as their username. (Required)
  • -p: Password to be associated with this new account. (Required)

7.2.4. List all the registered users

This command is used by the Cloud Administrator to get a list of all the registered users.

Usage: 

				
show users

7.2.5. Deactivate a user account

This command is used by the Cloud Administrator to make the account inaccessible, but does not delete any data.

Usage: 

				
deactivate -u <user account>

Command line options:

  • -u: Unique 'username' of the account being deactivated. (Required)

7.2.6. Activate a user account

This command is used by the Cloud Administrator to activate a user account. It makes the account accessible again if it were deactivated

Usage: 

				
activate -u <user account>

Command line options:

  • -u: Unique 'username' of the account being activated. (Required)
