12.1. Blackberry

The Blackberry platform by Research In Motion Limited is the first mobile platform supported by the OpenMobster Cloud Platform.

12.1.1. System Requirements

  • Java SE JDK v6.0
  • Eclipse 3.4 (the JDE plugin does not run on 3.5 yet)
  • Windows® 2000 SPI or later, or Windows XP
  • Windows Vista (BlackBerry JDE v4.2.1 and higher)
  • Monitor with a resolution of 1024 x 768 or higher

12.1.2. Some Helpful Development Tips

  • The SDK is available at: Blackberry SDK
  • OpenMobster has been developed using this environment: Blackberry JDE Plug-in for Eclipse v1.0 (with the BlackBerry JDE v4.5 Component Pack)
  • The platform version supported is version 4.3.0+. The Blackberry platform is fully backward compatible
  • Blackberry Development is officially supported only on the Windows Platform.
  • Blackberry follows a two-stage compilation process. The first stage called preverification (preverify) pre-verifies the Java code and produces the artifact to be processed by the next stage. The next stage called (rapc) actually produces the binary that can be installed on the device. The extension of the generated binary is .cod
  • Moblets/Apps that integrate with the OpenMobster Cloud Platform MUST combine their App's classes with the device-sdk-${project.version}-full.jar library before compiling their App into a binary. With the distribution, this library is located under: mobileCloud/blackberry-4.3.0/sdk
  • Blackberry has a selected set of APIs which require "signing" the binary (.cod) before installing it on the device. The OpenMobster binaries makes use of these APIs, and hence requires "signing" before installing it on the device. For more information about the "signing" process: Signing Process
  • Code Signing is not required for installing/testing Apps in a simulation environment. In fact all the OpenMobster binaries have been thoroughly tested on a 4.3.0 OS simulator. All tests have passed without any errors, warnings, or crashes.
  • As of 2.0-M3 community release, all OpenMobster Blackberry binaries have been signed and tested on an actual BlackBerry device. The device used was the BlackBerry Curve 8330 running on the Sprint CDMA network.
